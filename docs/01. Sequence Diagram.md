# Sequence Diagram

- Mermaid 라이브러리 Sequence Diagram 형식으로 작성

## 01. 로그인

```
sequenceDiagram
    participant UR as User
    participant EXT as Chrome Extension
    participant KKO as Kakao OAuth
    participant SVR as Server
    participant RMQ as Docker: RabbitMQ Consumer
    participant LMS as HYU LMS API
    participant DB as Docker: Supabase (PostgreSQL + Realtime)

    UR  ->> EXT: 로그인 시도
    EXT ->> KKO: 카카오 로그인
    KKO ->> EXT: 로그인 성공 (userId 포함)
    EXT ->> SVR: 유저 정보 요청 (/api/me)
    SVR ->> DB : 신규 유저 여부 확인

    alt 신규 유저 (status: needs_sync)
        SVR ->> EXT: { status: needs_sync }
        EXT ->> UR : LMS 동기화 안내 화면
        UR  ->> EXT: LMS로 이동 클릭
        EXT ->> UR : LMS 로그인 페이지 열기
        UR  ->> LMS: LMS 로그인
        LMS ->> EXT: 로그인 완료 감지 (tabs.onUpdated)
        EXT ->> DB : Realtime 구독 시작
        EXT ->> SVR: 동기화 요청 (세션 + 쿠키)
        SVR ->> RMQ: 동기화 요청
        RMQ ->> LMS: API 호출
        LMS ->> RMQ: API 응답
        RMQ ->> DB : 동기화 수행 및 저장 (SyncStatus = completed)
        DB  ->> EXT: Realtime push (변경 감지)
        EXT ->> UR : 동기화 완료 알림
    else 기존 유저 (status: ok)
        SVR ->> EXT: { status: ok }
        EXT ->> UR : 메인 화면
    end
```

## 02. 동기화

```
sequenceDiagram
    participant UR as User
    participant EXT as Chrome Extension
    participant KKO as Kakao OAuth
    participant SVR as Server
    participant RMQ as Docker: RabbitMQ Consumer
    participant LMS as HYU LMS API
    participant DB as Docker: Supabase (PostgreSQL + Realtime)

    UR  ->> EXT: 로그인 시도
    EXT ->> KKO: 카카오 로그인
    KKO ->> EXT: 로그인 성공(userId 포함)
    EXT ->> UR : 로그인 성공
    UR  ->> EXT: 동기화 버튼 클릭
    EXT ->> DB : Realtime 구독 시작 (sync_jobs 테이블)
    EXT ->> SVR: 동기화 요청
    SVR ->> RMQ: 동기화 요청
    RMQ ->> LMS: API 호출
    LMS ->> RMQ: API 응답
    RMQ ->> DB : 동기화 수행 및 저장 (status = completed)
    DB  ->> EXT: Realtime push (변경 감지)
    EXT ->> UR : 동기화 완료 알림
    EXT ->> DB : Realtime 구독 해제
```
